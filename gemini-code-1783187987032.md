# AI CONTEXT & TASK DEFINITION: ALPER AYHAN

## 1. PROJECT METADATA
*   **Project Name:** TKİ STYS Misafirhane Otomasyonu (MVP)
*   **Sprint Duration:** 14 Days (Week 1: Backend, Week 2: Frontend)
*   **Developer Role:** Team Lead, System Architect, API & Business Developer
*   **Goal:** Rapid MVP development focusing on core functionalities (Check-in, Check-out, Live Room Status, Invoicing).

## 2. TECHNOLOGY STACK
*   **Backend:** C# .NET 8 ASP.NET Core Web API
*   **Architecture:** N-Tier (Core, DataAccess, Business, API)
*   **Database:** Firebase Firestore (NoSQL - handled by teammates, interacted via Interfaces)
*   **Frontend:** Angular 18+ (Standalone Components), Tailwind CSS, PrimeNG
*   **Security:** JWT (JSON Web Token)
*   **Real-time Communication:** SignalR (WebSockets)
*   **Document Generation:** QuestPDF

## 3. WEEK 1 TASKS: BACKEND (C# .NET 8)
**Target:** Establish architecture, security, business logic, and real-time infrastructure.

*   **Architecture Setup:** Initialize the N-Tier solution (`TkiMisafirhane.sln`) with `API`, `Business`, `Core`, and `DataAccess` projects.
*   **Core Definitions:** Define foundational entities, DTOs (`RoomDto`, `GuestDto`, `InvoiceDto`), and Enums (`RoomStatus`, `GuestType`) in the `Core` layer.
*   **Interface Contracts:** Create interface structures (e.g., `IRoomRepository`, `IGuestRepository`) for the DataAccess developer to implement.
*   **Authentication:** Implement JWT Bearer authentication in the `API` layer and secure endpoints using the `[Authorize]` attribute.
*   **Business Logic (Billing):** Develop the invoice calculation algorithm (Nightly Rate x Stay Duration + Extras) within the `Business` layer.
*   **PDF Generation:** Integrate the `QuestPDF` library to generate official, A4-sized PDF invoices during the Check-out process.
*   **Real-time Hub:** Create a SignalR `RoomHub` in the `API` layer to broadcast room status changes (e.g., from Empty to Occupied) to all connected clients.
*   **Mock Repositories:** Create temporary Mock Repositories in the `API` layer for immediate testing while waiting for Firebase integration from teammates.
*   **Code Review:** Review, approve, and merge feature branches into the `develop` branch via GitHub Pull Requests.

## 4. WEEK 2 TASKS: FRONTEND (Angular 18+)
**Target:** Connect the Angular UI to the Backend API, manage application state, and ensure secure communication.

*   **Project Initialization:** Scaffold the Angular 18 project, configure `standalone` components setup, and install required libraries (PrimeNG, Tailwind, SignalR client).
*   **API Integration:** Write Angular `HttpClient` services (`RoomService`, `AuthService`, `BillingService`) to consume the C# endpoints.
*   **Frontend Security:** Implement an Angular HTTP Interceptor to automatically attach the JWT token to the headers of outgoing requests.
*   **Route Protection:** Develop an `AuthGuard` to prevent unauthorized access to the application routes.
*   **SignalR Client Integration:** Connect Angular to the backend WebSockets using `@microsoft/signalr` to listen for live room status updates.
*   **State Management:** Use RxJS (`BehaviorSubject` / `Signals`) to manage and distribute real-time data across the UI components dynamically.
*   **Data Binding:** Inject the API services into the UI components/forms (developed by teammates) to replace hardcoded mock data with real database interactions.

## 5. DEVELOPMENT RULES FOR AI ASSISTANCE
*   **Strict Adherence:** AI must provide solutions strictly within the defined technology stack and architecture.
*   **No Code Over-engineering:** Solutions must focus on MVP requirements (working, reliable, fast to implement). Refactoring for extreme performance is out of scope for the 14-day sprint.
*   **Contextual Awareness:** AI must assume Alper has access to the `API`, `Business`, and `Core` layers, but does NOT write the Firebase `DataAccess` code directly. AI should use Dependency Injection to access data.
